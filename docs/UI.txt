================================================================
  NOTION AUTO PAY — ОПИСАНИЕ ИНТЕРФЕЙСА (веб-панель)
================================================================

Адрес панели:  http://localhost:8081/dashboard/
Тема:        тёмная (чёрный фон, светлый текст, фирменный mesh-градиент)
Адаптив: полная мобильная версия

Этот файл описывает все экраны, кнопки и элементы управления панели.


================================================================
0. ЭКРАН ВХОДА (появляется, только если задан пароль)
================================================================

Показывается, когда сервер запущен с паролем
(build.bat --password=… или переменная DASHBOARD_PASSWORD).
Без пароля этот экран пропускается.

Элементы:
  • Логотип «Notion Auto Pay» с иконкой-искрой.
  • Заголовок «Вход в панель».
  • Поле ввода пароля (скрытый ввод, фокус автоматически).
  • Кнопка «Войти» (во время проверки показывает «Проверка...»,
    неактивна при пустом поле).
  • При ошибке — красный текст «Неверный пароль».

Пароль никогда не уходит на сервер в открытом виде — хешируется в браузере.
После входа в шапке появляется кнопка «Выйти».


================================================================
1. ШАПКА (Hero, верхняя полоса)
================================================================

Верхняя полоса с атмосферным mesh-градиентом. Содержит:

Правый верхний угол:
  • Кнопка «+ Добавить аккаунт» (белая) — открывает модальное окно
    добавления аккаунта по token_v2.
  • Кнопка «Выйти» (тёмная, с рамкой) — только когда включён пароль.

Центр:
  • Плашка-ярлык «✦ Notion Auto Pay».
  • ПЕРЕКЛЮЧАТЕЛЬ ВКЛАДОК (капсула из двух кнопок):
        [ Оплата ] [ Чат ]
      Активная вкладка — белая с чёрным текстом.

Ниже (если есть аккаунты) — два счётчика:
  • Число аккаунтов
  • Число пространств


================================================================
2. МОДАЛЬНОЕ ОКНО «ДОБАВИТЬ АККАУНТ»
================================================================

Открывается кнопкой «+ Добавить аккаунт». Содержит:
  • Заголовок «Добавить аккаунт Notion» + крестик × (закрыть).
  • Инструкция, как получить token_v2
    (notion.so -> F12 -> Application -> Cookies -> token_v2).
  • Поле (textarea) для вставки token_v2.
  • Кнопки внизу:
        [ Отмена ]      — закрыть без сохранения.
        [ Добавить аккаунт ] — проверяет токен, автоматически
                            находит все пространства аккаунта.

После успеха показывает карточку «Аккаунт добавлен» (имя, e-mail,
пространство, тариф) и через ~1.5 с закрывается.
Закрыть также можно клавишей Escape или кликом по фону.


================================================================
3. ВКЛАДКА «ОПЛАТА»
================================================================

Если аккаунтов ещё нет — показывается пустое состояние
«Пока нет рабочих пространств» с кнопкой «+ Добавить аккаунт».

--- 3.1 Шапка списка пространств -----------------------------

Строка «Рабочие пространства (N)» и рядом две иконки:
  • Иконка ОБНОВЛЕНИЯ (круговая стрелка) — перезапрашивает
    тарифы и список пространств (только обновляет данные,
    ничего не оплачивает). Крутится во время загрузки.
  • Иконка ШЕСТЕРЁНКИ — открывает попап настроек автооплаты.

Справа от иконок — подписи:
  • «авто · Nс» — если автооплата включена (показывает интервал).
  • «обновлено ЧЧ:ММ:СС» — время последнего ручного обновления.

--- 3.2 Попап настроек автооплаты (иконка шестерёнки) -------

Заголовок «Автооплата (на сервере)» — работает в фоне,
браузер можно закрыть. Элементы:
  • Тумблер «Платить при Free тарифе» — вкл/выкл автооплаты.
    Неактивен, пока не задана карта.
  • Поле «Интервал проверки, сек» — числовое (мин. 5, макс. 86400).
  • Строка «План: …» — выбранный тариф для оплаты.
  • Строка «Карта: ···· 1234» или «не задана».
  • Кнопка «Настроить карту и план» — открывает окно сохранения
    карты/выбора плана.
  • Кнопка «Проверить и оплатить сейчас» — ручной запуск
    сканирования (неактивна без карты/выкл. автооплаты).
  • Блок «Последние автооплаты» — лог последних событий.

Предупреждение: каждое Free-пространство оплачивается один раз,
списываются реальные деньги.

--- 3.3 Карточка аккаунта -------------------------------------

Каждый аккаунт — отдельная карточка:
  • Круглый аватар (первая буква имени).
  • Имя и e-mail + число пространств.
  • Кнопка-МЕНЮ (три точки) справа:
        - «Скопировать токен аккаунта»
        - «Убрать из списка» (красным)

--- 3.4 Карточка пространства (внутри аккаунта) ----------

Пространства идут сеткой (1 колонка на телефоне, 2 на широком
экране). В каждой:
  • Иконка пространства (эмодзи/картинка/первая буква).
  • Название + ЦВЕТНОЙ БЕЙДЖ тарифа (Free / Plus / Team /
    Business / Enterprise — разные цвета).
  • Строка «Подписка: …» или «Бесплатный план» (+ домен).
  • Чекбокс «Авто» — включает автооплату именно этого пространства.
  • Кнопка «Оплатить» (белая) — открывает окно оплаты
    для этого пространства.
  • ПОЛОСА AI-ТОКЕНОВ (если есть лимит) — «осталось X% · N из M»,
    цвет меняется: зелёный > 50%, жёлтый > 20%, красный ниже.

--- 3.5 Окно оплаты (SubscribeModal) -------------------------

Открывается кнопкой «Оплатить». Позволяет выбрать план,
ввести данные карты и оплатить, либо оплатить сохранённой картой.
(Отдельный компонент SubscribeModal.)

--- 3.6 Окно настройки карты (AutoPaySettings) ---------------

Открывается кнопкой «Настроить карту и план». Сохраняет карту
на сервере для фоновой автооплаты и выбора тарифа.


================================================================
4. ВКЛАДКА «ЧАТ»
================================================================

Чат с AI-агентом Notion по каждому пространству. Две зоны:
боковая панель слева и область диалога справа.

--- 4.1 Боковая панель (слева) ----------------------------

  • Выпадающий список «Пространство» — выбор рабочего
    пространства (показываются только платные или с автооплатой).
  • Выпадающий список «Агент» — обычный или кастомные агенты.
  • Кнопка «+ Новый чат» (синяя) — начать новый диалог.
  • Раздел «История» — список прошлых чатов. У каждого при
    наведении появляется иконка КОРЗИНЫ (удалить чат).
    Активный чат подсвечен.

--- 4.2 Мобильная версия боковой панели ----------------

На телефоне боковая панель спрятана в выдвижное меню (drawer).
Сверху области чата появляется мобильная панель:
  • Кнопка «☰» (гамбургер) — открывает drawer со списком чатов.
  • Название текущего чата по центру.
  • Кнопка «+» — новый чат.
Drawer закрывается крестиком ×, кликом по затемнённому фону
или после выбора чата.

--- 4.3 Область диалога -------------------------------------

Сообщения оформлены в стиле DeepSeek/ChatGPT:
  • Сообщение ПОЛЬЗОВАТЕЛЯ — компактный пузырь справа.
  • Ответ АГЕНТА — на всю ширину, с аватаром-точкой «✦» слева,
    без тяжёлого пузыря.
  • Над ответом — сворачиваемый блок «Шаги агента · N»:
        - инструменты (с иконкой, напр. «GitHub / get_me»),
          при клике раскрывают Input и Response;
        - «Размышления» (💡) — раскрываются в полный текст.
  • Под ответом — кнопка «Копировать».
  • Блоки кода имеют свою кнопку копирования в углу; длинный
    код/таблицы скроллятся внутри себя (страница больше не
    листается вправо).
  • При прокрутке вверх подгружаются старые сообщения.

СОСТОЯНИЯ ОТВЕТА (теперь понятно, что происходит):
  1. «Думаю…» — крутится спиннер + три пульсирующие точки,
     пока агент думает и ещё нет текста.
  2. «Печатает…» — текст появляется вживую с мигающим
     курсором (синяя палочка).
  3. ГОТОВО — курсор исчезает, сообщение фиксируется.

Пустой чат показывает подсказку «Напишите сообщение…».

--- 4.4 Поле ввода (композер, внизу) ----------------------

  • Текстовое поле АВТОРАСТЁТ по высоте под длинное сообщение
    (до ~220px, дальше появляется внутренняя прокрутка).
  • Выпадающий список модели (только для обычного агента,
    по умолчанию Opus).
  • Круглая кнопка ОТПРАВКИ (иконка самолётика).
  • Подсказка: «Enter — отправить · Shift+Enter — новая строка».

Горячие клавиши:
  • Enter        — отправить
  • Shift+Enter  — перенос строки


================================================================
5. ТЕХНИЧЕСКИЕ ДЕТАЛИ
================================================================

  • Автооплата выполняется на сервере (фоновый планировщик),
    даже при закрытом браузере.
  • Чат проксирует приватный протокол Notion AI по token_v2
    каждого аккаунта; текст ответа стримится вживую.
  • Панель — React-SPA, встраивается в Go-бинарник при сборке
    (build.bat: npm run build -> internal/web/dist -> go run).
  • Порт по умолчанию :8081.
